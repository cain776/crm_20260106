## 🔍 분석 및 인사이트

### 주요 특징

| 특징 | 설명 |
|------|------|
| **병원 관리 세분화** | 태그 시스템(환자/접수/상태)으로 상세 분류 지원 |
| **외부 연동 강조** | 네이버예약, 구글 비즈, 여신티켓 등 파트너 연동 |
| **보안 강화** | IP 제한, 권한 설정 등 보안 기능 별도 메뉴화 |
| **CRM 통합** | 메시지 발송, 만족도 조사 등 환자 소통 기능 |
| **라이브러리화** | 수가/약가/오더세트 등 재사용 가능한 설정 분리 |

### 메뉴 구조 통계

| Depth | 메뉴 수 |
|-------|--------|
| Level 1 (대분류) | 8개 |
| Level 2 (중분류) | 16개 |
| Level 3 (소분류) | 35개 |
| **총계** | **59개** |

### 안과 CRM에 참고할 점

1. **태그 시스템**: 환자태그, 접수태그, 상태태그로 환자 분류 → 안과에서 수술유형별, VIP등급별 태그 적용 가능
2. **예약 연동**: 네이버예약 연동 → 굿닥, 똑닥 등 안과 예약 플랫폼 연동 고려
3. **만족도 조사**: 별도 메뉴로 분리 → 수술 후 만족도 추적에 활용 가능
4. **라이브러리**: 오더세트 개념 → 안과 검사 세트(굴절, 망막, 각막 등) 구성에 참고
5. **부가기능 연동**: 외부장비(카메라) → 안과 검사장비 연동 확장 가능
